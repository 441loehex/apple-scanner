# Known Limitations — Apple Caliber Scan

**Freshora Sp. Z. o. o.**

This document describes known limitations of the system. Understanding these limitations is
essential for correct interpretation of results.

---

## 1. Top Layer Only

**The system measures only the visible top layer of each crate.**

LiDAR and photogrammetry cannot see through apples. Apples in lower layers are invisible
to the scanner. Reports always state "warstwa wierzchnia" (top layer) and must not be
interpreted as a measurement of the entire crate.

---

## 2. Batch Projection Is a Heuristic Estimate

Whole-batch weight and caliber projections are computed by assuming the top layer is
representative of all layers. This assumption is:

- **Often reasonable** for mechanically sorted batches
- **Unreliable** when top layers are hand-selected or contain a mix of sizes
- **Always labeled SZACUNEK (estimate)** in reports

Do not use batch projections for legal or commercial weight/grade documentation.

---

## 3. Caliber Accuracy Depends on Ring Calibration

If the measuring ring is not annotated in the scan review step, the system cannot compute
a physical scale factor. Caliber values will be labeled **BRAK KALIBRACJI** (uncalibrated)
and diameters are in arbitrary pixel units — not millimeters.

Even with a calibrated ring:
- Measurement error is approximately ±3–5 mm per apple at normal scanning height
- Apples near the edge of the crate may be partially occluded → underestimated diameter
- Tilted or non-spherical apples may show biased measurements

---

## 4. Circle Detection Is Not Perfect

The Hough circle detector identifies candidate circles in the top-down density projection.
Detection can fail or produce false positives when:

- Apples overlap significantly (small caliber, dense packing)
- Lighting is non-uniform, creating dark shadows that break the circle boundary
- Scan density is low due to fast or distant scanning
- Non-circular objects are present (stems, leaves, packaging)

All automatically detected circles should be reviewed in the annotation canvas before
generating a final report.

---

## 5. Confidence Levels

| Level | Polish | Meaning |
|-------|--------|---------|
| HIGH / wysoka | Only when grader ground truth is entered | Statistical validation available |
| LOW / niska | Calibrated ring present, no grader GT | Physical scale known, no validation |
| UNCALIBRATED / brak kalibracji | No ring annotated | Pixel units only |

**HIGH confidence is rare.** It requires a grader printout to be entered via the ground-truth
import feature. Most field scans will produce LOW confidence results.

---

## 6. Weight Model

The per-apple weight estimate uses a geometric heuristic:

```
weight_g ≈ 0.0005 × (diameter_mm / 2)³
```

This is a rough geometric sphere model. Actual apple weight varies by:
- Variety (density, shape factor)
- Water content / storage condition
- Stem/calyx presence

Weight estimates carry ±20–30% error and are provided for rough planning only.

## 7. File Format Support

| Format | Support Level |
|--------|--------------|
| PLY | Full (point cloud) |
| OBJ | Full (mesh, vertices extracted) |
| GLB/GLTF | Full (mesh, via trimesh) |

| USDZ | Accepted, not processed — returns empty point cloud |

---

## 8. Reports Are Not Certified

Reports generated by this system are **not**:
- Certified by any Polish or EU standards body
- Equivalent to machine-sorter output
- Legal documentation for customs, phytosanitary, or retail compliance

Reports include a mandatory legal clause (klauzula wyłączenia odpowiedzialności) to this
effect. Do not remove this clause from exported documents.

---

## 9. Google Drive Dependency for Scan Delivery

The system downloads scan files from Google Drive. This requires:
- A Google account with the file shared as "anyone with the link"
- Internet access on the machine running the web server
- Files under ~2 GB (gdown limit)

Very large scans (>200 MB PLY) may process slowly and risk timeout on low-memory machines.

---

## 10. No Real-Time or Multi-Crate Scanning

The system processes one scan per crate per batch entry. It does not support:
- Live camera feeds
- Automatic multi-crate workflows
- Robot or conveyor integration

---

## 11. Single-User Web UI

The web UI uses cookie-based sessions with a single shared password (`ACS_WEB_PASSWORD`).
It is not designed for concurrent multi-user access. Use separate deployments or batch
numbers to separate operator workflows.
